# 0001. ウィルパワーは暦日変更で自動リセットしない

- 日付: 2026-07-09
- 状態: 採用

## 背景

Issue #33「意思力の時間経過に伴う扱いを仕様化する」への対応。現状、ウィルパワーは消費/回復/リセットの手動操作でのみ変化し、日付が変わっても自動では変化しない。ユーザーテストを実施する前に、この挙動を意図した仕様として確定させる必要があった。

一般的な「デイリーカウンター」系アプリでは深夜0時に自動リセットする実装がよく見られるため、WillMeterでも同様の対応が必要か検討した。

## 選択肢

1. 深夜0時に自動で最大値までリセットする（デイリーカウンター方式）
2. 時間経過に応じて徐々に回復させる（例: 1時間ごとに一定量）
3. 日付変更による自動変化は行わず、ユーザーの明示的な操作（消費/回復/リセット）でのみ状態が変わる

## 採用案

選択肢3。日付変更・時間経過による自動変化は実装しない。

## 採用理由

WillMeterのウィルパワーは「暦日」ではなく「睡眠・休息などの回復行動」によって回復するという概念を採用しているため。徹夜した場合に日付が変わっただけで意思力が満タンに戻るのは、コンセプト上不自然。ユーザーが眠っていなくても0時を過ぎた瞬間に回復して見えるのは体験として誤りである。

## 欠点・リスク

- ユーザーが自分でリセット操作を忘れると、実態と表示がずれたまま長期間放置される可能性がある
- 「回復行動の記録」に基づく自動回復（例: 睡眠カテゴリのタスク記録をトリガーにする）は将来的に検討する価値があるが、現状はタスク管理UI自体が未実装（README/CLAUDE.mdのMVPスコープ外、#28参照）のため対象外とした

## 再評価条件

- タスク管理機能（#28で言及したロードマップ項目）を実装し、「睡眠」等のカテゴリと連動した回復ロジックを導入する場合
- ユーザーテストで「日を跨ぐと表示が実態と合わなくなる」というフィードバックが継続的に得られた場合

## 影響範囲

- 現時点でコード変更なし（`WillPower`/`WillPowerViewModel`に日付依存ロジックを追加しない）
- 将来、タスク管理機能実装時にこのADRを参照し、回復ロジックの設計判断に活用する
